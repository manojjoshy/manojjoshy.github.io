Designing the Admin Portal for Nuvepro Cloud Labs

Role: UX designer, UI designer
Date: Sept 2022 - July 2023
Team: Adarsh EA, Mohd. Zuheb, Manoj Joshi, Pooja Kannan
Tools: Adobe XD, Microsoft Teams, Zoho Sprints, Sharepoint, draw.io

Overview

Nuvepro Technologies required an admin portal to manage their new LMS, focusing on designing a user-friendly interface for course creation, lab integration, and overall management. The portal needed to accommodate three primary user roles: administrators, managers, and learners. The aim was to streamline the process of creating and managing courses, integrating labs, and overseeing user interactions.

Background

Nuvepro Technologies provides cloud-based and virtual lab environments for various educational and enterprise needs. The admin portal was designed to support administrators in creating and managing educational content, integrating sandbox environments, and facilitating user management within the LMS.

Project Goals

Efficient Course Management

Create an intuitive interface for administrators to design and manage courses, including content and associated labs.

Seamless Lab Integration

Provide tools to integrate both cloud-based labs and virtual machines into course modules.

User Management

Implement features for managing user roles and permissions, tracking progress, and assessing performance.

Design Process

User Research and Analysis

During the user research and analysis phase, we identified two primary personas: Emily Johnson, an administrator responsible for course creation and lab integration, and John Smith, a manager overseeing learner progress. Through interviews and task analyses, we uncovered their distinct goals and pain points. Emily required an intuitive interface to create courses efficiently, integrate labs seamlessly, and access actionable analytics, while John needed tools to track learner performance, customize reports, and manage learning paths effectively. These insights guided our design decisions, ensuring the portal addressed their unique needs and workflows.

      Stakeholder Interviews

      To ensure the admin portal aligned with both business objectives and user expectations, we conducted stakeholder interviews with Nuvepro’s key teams, including product managers, LMS administrators, and technical leads. These discussions provided valuable insights into the operational challenges faced by administrators and managers, such as managing diverse lab environments and tracking user performance. Stakeholders emphasized the need for an intuitive interface, robust reporting tools, and seamless lab integration capabilities. Their input played a critical role in defining project priorities and shaping the portal's features to balance usability with technical feasibility.

      Business Goals

      • Streamline LMS Operations: Develop a user-friendly admin portal that simplifies course creation, lab integration, and user management to enhance operational efficiency.
      • Increase Adoption: Provide an intuitive interface that encourages frequent use by administrators and managers, driving higher engagement with the LMS.
      • Enhance Analytics Capabilities: Offer robust reporting and analytics tools to demonstrate value to educational institutions and enterprise clients.

      User Needs

      • Efficient Course Management: Administrators need a straightforward way to create and manage courses, integrate labs, and configure learning paths.
      • Accessible Reporting Tools: Managers require customizable, easy-to-navigate reports to track learner progress and outcomes effectively.
      • Clear and Intuitive UI: Both roles need an interface that minimizes complexity and supports seamless navigation across functionalities.

      Technical Requirements

      • Lab Integration Tools: Enable seamless integration with cloud-based and virtual lab environments such as AWS, Azure, and GCP.
      • Scalable Architecture: Ensure the system supports growing user bases, varied course sizes, and complex lab setups without performance degradation.
      • Secure Role-Based Access Control: Implement robust authentication and authorization mechanisms to manage user roles and permissions securely.

      User Flows

      We created detailed user journeys to map out the interactions of our primary personas, Administrator and Manager, with the admin portal. These journeys outlined key tasks, such as Administrator’s process for creating courses and integrating labs, and Manager’s workflow for tracking learner progress and generating reports. By identifying touchpoints and potential pain points, such as complex navigation or insufficient reporting tools, we ensured the design addressed user frustrations while supporting their goals. These user journeys provided a clear framework for crafting an intuitive and efficient user experience.

      Design Strategy & Implementation

      Typography & Colors

      We chose Poppins as the primary font to create a modern, clean, and approachable interface. Poppins' geometric sans-serif style provides excellent readability across devices, ensuring both learners and administrators can easily navigate content. Its rounded shapes and balanced proportions convey a friendly yet professional tone, aligning well with Nuvepro’s goal of being both user-centric and functional. The font's versatility, with a wide range of weights, allows for clear hierarchy in the design, from bold headlines to lighter body text, enhancing overall visual clarity and user experience.

      Icon Design


      We utilized the IcoMoon Ultimate icon font to ensure a cohesive and professional visual language. The icons were carefully selected to complement the design aesthetics and enhance usability, providing clear, intuitive cues for actions and navigation. By leveraging IcoMoon’s versatility and compatibility, we seamlessly integrated the icons into the interface, aligning them with the existing design elements while maintaining visual consistency across the portal. This approach improved user comprehension and streamlined interactions.

      Component Design


      Our component design focused on creating intuitive and reusable elements for form inputs and navigation. For the form elements, we designed a clean and structured layout with dropdowns, text fields, and checkboxes to guide users through complex tasks like course creation and lab integration. Navigation components, including a side menu and breadcrumb trails, ensured seamless access to different sections of the portal, enhancing usability and reducing cognitive load. These components were crafted with scalability in mind, ensuring they could adapt to future features while maintaining a consistent user experience.

      Medium Fidelity Wireframes

      Testing and Iteration

      During the iteration phase, we refined the course creation process to align with user needs and expectations. A course was defined as a structure composed of topics, where each topic contained activities—the smallest unit within the course. Activities could include a combination of cloud labs, multimedia content, or assignments. By breaking courses into these modular components, administrators gained flexibility in designing learning paths, making it easier to integrate labs or adjust content dynamically. This hierarchical approach ensured clarity and streamlined course management.

      Additionally, we focused on enhancing the management of groups and cohorts. Groups allowed administrators to organize learners based on teams, departments, or skill levels, while cohorts enabled the assignment of specific learning paths to these groups. This functionality facilitated targeted learning experiences, progress tracking, and personalized resources. By iterating on these features, we created an intuitive system for managing learners at scale, enabling tailored educational strategies for diverse user bases.

      Further Additions

      Course Creation

      Steps in Course Creation:

      • Define Course Basics
        • Title, description, and tags for searchability.
        • The form also allowed admins to select whether the course would be public (for independent learning providers) or private (for internal teams).
      • Upload Course Materials:
        • Administrators could upload files, embed videos, or add external links using an easy-to-navigate media manager.
        • Courses could be sectioned into multiple parts using drag-and-drop elements to arrange content.
      • Select Lab Environments:
        • This step integrated lab environments seamlessly into the course. Admins could choose from existing playgrounds or challenge labs or create new ones on the fly.
        • They could also configure specific parameters, such as access duration for labs, sandbox persistence, and cloud provider options.
      • Skill Bundles & Progression Settings:
        • Admins could group content into skill bundles, ensuring learners completed prerequisite modules before progressing.
        • A course map visualized the entire structure, helping admins review the flow of playgrounds and challenge labs.
      • Assessment Configuration:
        • Administrators could set up assessments for challenge labs, including passing criteria, auto-grading options, and manual review for tasks that required subjective evaluation.
      • Preview Mode:
        • Before publishing, the admin could preview the entire course and lab environments from the learner’s perspective to ensure all content and settings were correct.
